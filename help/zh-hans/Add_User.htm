添加用户

利用“添加用户”页面可以创建新的 Multi-Factor Authentication 帐户。

 

注意： 根据 Multi-Factor Authentication 服务器的“公司设置”部分中指定的用户默认设置，许多用户设置，如方法（电话呼叫、短信或移动应用程序）都是默认的。

 

若要添加新用户：

  1. 输入用户的详细信息。

  2. 单击“添加”按钮。

 

功能

描述

用户名

用户名通常是具有正确电子邮件格式的用户电子邮件。

 

示例： chuck@abc.com

 

名字

输入用户的名字。

 

姓氏

输入用户的姓氏。

 

电子邮件地址

输入用户的电子邮件地址。 电子邮件将发送到此电子邮件地址。 如果未指定，则电子邮件将发送到具有正确电子邮件格式的用户名。

 

用户组

输入用户的用户组。

 

国家/地区代码

从下拉列表中选择相应的国家/地区代码。默认设置为“美国和加拿大 +1”。

 

注意：某些国际电话呼叫需要收取费用。 如果您的公司尚未实施此选项，则系统不会向某些国家/地区代码发起呼叫。

 

电话号码

输入身份验证期间 Multi-Factor Authentication 呼叫或发送短信时将使用的电话号码。这有可能是移动电话号码、座机号码，或者是电话呼叫方法中的其他一些号码。 对于短信方法，电话必须具备短信功能。 电话号码的格式必须与“国家/地区代码”字段中选定的格式一致。

 

示例：对于美国和加拿大的电话号码，必须输入 10 位数字的电话号码（例如 555-123-4567）。

 

分机号

输入在呼叫应答后将要拨打的分机号。 允许使用数字、逗号、* 和 #。 逗号可用于暂停一秒钟以导航电话系统菜单。 此为可选字段。

 

备份国家/地区代码

请选择用于备份电话号码的相应国家/地区代码。

 

备份电话号码

输入在呼叫主要电话没有应答时，Multi-Factor Authentication 将要呼叫的电话号码。

 

备份分机号

输入备份电话号码的分机号。

 

PIN

键入用户的 PIN。 或者，您可以通过单击“生成”按钮生成 PIN。 PIN 必须符合“PIN 最小长度”和“弱 PIN”规则。

 

“生成”按钮

“生成”按钮提供一个随机生成的 PIN。 所生成的 PIN 的长度将会大于 4 位数字或大于用户的最小 PIN 长度。

 

确认 PIN

重新键入你在 PIN 字段中输入的 PIN。确认 PIN 必须完全匹配。

 

已启用

指定是否启用该用户。 默认为选中状态。

 

注意：如果没有启用用户记录，则用户在登录时将不会收到 Multi-Factor Authentication 呼叫，并且根据公司对禁用行为的默认设置，可能允许或禁止用户访问应用程序。

 

需要更改 PIN

指定用户在下一次 Multi-Factor Authentication 呼叫期间必须更改其 PIN。

 

注意：只有在公司配置允许用户更改其 PIN 的情况下，才会显示“需要更改 PIN”复选框。

 

发送电子邮件

指定是否应将一封电子邮件发送到已添加的用户。

 

注意：只有当公司配置了电子邮件时，才会显示“发送电子邮件”复选框。
