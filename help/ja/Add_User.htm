ユーザーの追加

[ユーザーの追加] ページでは、新しい Multi-Factor Authentication アカウントを作成できます。

 

注: 認証方法 (電話、テキスト メッセージ、モバイル アプリ) など、多くのユーザー設定は、Multi-Factor Authentication Server の [会社設定] に指定されたユーザーの既定に基づいて、既定が指定されます。

 

新しいユーザーを追加するには:

  1. ユーザーの詳細を入力します。

  2. [追加] ボタンをクリックします。

 

機能

説明

ユーザー名

通常、ユーザー名は、正しい電子メール アドレス形式のユーザーの電子メール アドレスです。

 

例: chuck@abc.com

 

名前 (名)

ユーザーの名前 (名) を入力します。

 

名前 (姓)

ユーザーの名前 (姓) を入力します。

 

電子メール アドレス

ユーザーの電子メール アドレスを入力します。 電子メールはこの電子メール アドレスに送信されます。 指定しない場合、ユーザー名が正しい電子メール アドレス形式であれば、電子メールはユーザー名に送信されます。

 

ユーザー グループ

ユーザーのユーザー グループを入力します。

 

国コード

ドロップダウン リストから適切な国コードを選択します。既定は、[米国およびカナダ +1] に設定されています。

 

注:国際電話には料金がかかる可能性があります。 会社がこのオプションを実装していない場合、一部の国コードに対する電話は発信されません。

 

電話

Multi-Factor Authentication が認証中に電話をかけるかテキストを送信する電話番号を入力します。電話による認証方法として、携帯電話番号、固定電話番号、またはその他の番号を指定できます。 テキスト メッセージによる認証方法を使用するには、テキスト メッセージ機能を持つ電話が必要です。 電話番号の形式は、[国コード] フィールドで選択した形式と一致する必要があります。

 

例:米国とカナダの電話番号の場合、10 桁の電話番号を入力する必要があります (555-123-4567 など)。

 

内線

電話がつながった後にダイヤルする内線番号を入力します。 数字、コンマ、*、# だけを使用してください。 電話システムのメニューに移動するために 1 秒間途切れさせるにはコンマを使用します。 このフィールドはオプションです。

 

予備の国コード

予備の電話番号に適切な国コードを選択します。

 

予備の電話番号

メインの電話が応答しない場合に Multi-Factor Authentication が発信する電話番号を入力します。

 

予備の内線

予備の電話番号の内線を入力します。

 

PIN

ユーザーの PIN を入力します。 または、[生成] ボタンをクリックして、PIN を生成できます。 PIN は、[PIN の最小の長さ] および [脆弱な PIN] の規則に従う必要があります。

 

[生成] ボタン

[生成] ボタンをクリックすると、PIN がランダムに生成されます。 4 桁またはユーザーの [PIN の最小文字数] のうち、長い方に合わせて PIN が生成されます。

 

PIN の確認入力

[PIN] フィールドに入力した PIN を再入力します。最初に入力した PIN と正確に同じでなければなりません。

 

有効

ユーザーを有効にするかどうかを指定します。 このチェック ボックスは既定でオンになっています。

 

注: ユーザー レコードが有効ではない場合、そのユーザーは、サインオン時に Multi-Factor Authentication の電話を受けません。また、無効な動作に関する会社の既定に応じて、アプリケーションへのアクセスが許可または拒否される可能性があります。

 

PIN の変更が必要

ユーザーが次の Multi-Factor Authentication の電話で PIN を変更する必要があることを指定します。

 

注: [PIN の変更が必要] チェックボックスが表示されるのは、会社の構成で、PIN の変更がユーザーに許可されている場合のみです。

 

電子メールの送信

追加したユーザーに電子メールを送信するかどうかを指定します。

 

注: [電子メールを送信] チェックボックスが表示されるのは、会社の電子メールが構成されている場合のみです。
