Настройка пользователя

Если настройка учетной записи не завершена, то необходимо указать номер телефона (если применимо) и/или ПИН-код (если применимо), которые необходимо использовать для учетной записи, или может быть выдан запрос на создание кода активации, который служит для активации приложения Multi-Factor Authentication.  Если компания разрешает это, можно выбрать один из следующих способов: телефонный вызов, SMS, мобильное приложение и OATH-токен.  Сервер Multi-Factor Authentication сделает телефонный вызов или отправит SMS на указанный номер телефона, чтобы подтвердить его действительность, направит уведомление в мобильное приложение или выдаст запрос на ввод кода проверки подлинности.  Если выбран способ, связанный с использованием телефонного вызова, пользователя попросят нажать кнопку #, ввести свой ПИН-код или зарегистрировать свой голосовой отпечаток (в зависимости от режима телефонного вызова).  Если выбран способ, связанный с использованием SMS, пользователю будет направлен одноразовый секретный код (OTP), требующий ответа в виде аналогичного кода OTP или кода OTP с указанием ПИН-кода.  Если выбран способ, связанный с использованием мобильного приложения, то на мобильное приложение будет отправлено уведомление с запросом на проверку подлинности, которое может требовать от пользователя ввода ПИН-кода (в зависимости от действующего режима мобильного приложения).  Если выбран способ, связанный с использованием OATH-токена, то пользовательский портал выдаст пользователю запрос на ввод кода проверки подлинности.  В случае успешного завершения для пользователя может отобразиться запрос на создание контрольных вопросов.

 

Чтобы завершить настройку способа телефонного вызова для пользователя, выполните следующие действия.

  1. Введите код страны и номер телефона.  Можно также ввести резервный телефонный номер (если применимо).

  2. Введите ПИН-код (если применимо).

  3. Нажмите кнопку Позвонить для проверки подлинности.

 

Чтобы завершить настройку отправки текстовых сообщений пользователю, выполните следующие действия.

  1. Введите код страны и номер телефона.

  2. Введите ПИН-код (если применимо).

  3. Нажмите кнопку Отправить текстовое сообщение для проверки подлинности.

 

Чтобы завершить для пользователя настройку способа проверки подлинности на основе мобильного приложения, выполните следующие действия.

  1. Установите мобильное приложение на телефон.

  2. Нажмите кнопку Создать код активации.

  3. Введите в мобильное приложение код активации и URL-адрес, чтобы завершить активацию.

  4. Введите ПИН-код (если применимо).

  5. Нажмите кнопку Проверить подлинность пользователя.

 

Чтобы завершить для пользователя настройку способа проверки подлинности на основе токена OATH, выполните следующие действия.

  1. Установите мобильное приложение на телефон.

  2. Нажмите кнопку Создать код активации.

  3. Введите в мобильное приложение код активации и URL-адрес, чтобы завершить активацию.

  4. Нажмите кнопку Проверить подлинность пользователя.

ИЛИ

  1. Введите серийный номер, напечатанный на тыльной стороне токена OATH.

  2. Введите код, отображаемый на токене OATH.

  3. Нажмите кнопку Проверить токен OATH.

Метод

Выберите один из следующих методов: телефонный вызов, текстовое сообщение, мобильное приложение или токен OATH.  Если выбран способ, связанный с использованием телефонного вызова, пользователь получает телефонный вызов от сервера Multi-Factor Authentication с просьбой нажать кнопку #, ввести свой ПИН-год или зарегистрировать голосовой отпечаток.  Если выбран способ, связанный с использованием SMS, пользователю будет направлено текстовое сообщение с одноразовым секретным кодом (OTP), требующее ответа в виде аналогичного кода OTP или кода OTP с указанием ПИН-кода.  Если выбран способ, связанный с использованием мобильного приложения, то на мобильное приложение будет отправлено уведомление с запросом на проверку подлинности, которое может требовать от пользователя ввода ПИН-кода.  Если выбран способ, связанный с использованием OATH-токена, то пользовательский портал выдаст пользователю запрос на ввод кода проверки подлинности.

 

Примечание.    Выбранный метод будет использоваться для доступа к приложениям, защищенным сервером Multi-Factor Authentication.  Этот метод можно изменить в любой момент на странице Change Method (Изменение метода).

 

Код страны

В раскрывающемся списке выберите код соответствующей страны. Если для учетной записи уже настроен номер телефона, то по умолчанию используется код страны для этого номера телефона.  По умолчанию для США и Канады установлен код +1.

 

Примечание. Некоторые международные вызовы тарифицируются.  Если системный администратор не реализовал поддержку этого варианта, то телефонные вызовы не будут совершаться на некоторые коды стран.

 

Телефон

Введите номер телефона, на который сервер Multi-Factor Authentication сделает вызов или направит текстовое сообщение в ходе проверки подлинности. Этот номер может принадлежать мобильному телефону, настольному телефону или иному переговорному устройству.  Это устройство должно представлять собой телефон с поддержкой приема и отправки SMS в режиме работы с SMS.  Формат телефонного номера должен совпадать с форматом, выбранным в поле Country Code (Код страны).

 

Пример. Для телефонных номеров в США и Канаде необходимо ввести 10-значный номер телефона (например, 555-123-4567).

 

Добавочный номер

Введите добавочный номер, который будет набираться после ответа на звонок.  Разрешены цифры, запятые, знаки "*" и "#".  Запятые могут использоваться для секундной паузы при переходе по системным меню телефона.  Это поле является необязательным.

 

Код страны для резервного номера телефона

Выберите код соответствующей страны для резервного номера телефона.

 

Резервный номер телефона

Введите номер телефона, на который Multi-Factor Authentication сделает вызов в случае, если по основному номеру телефона не будет получен ответ.

 

Добавочный номер резервного телефона

Введите добавочный номер для резервного телефона.

 

ПИН-код

Введите ПИН-код.  Чтобы завершить проверку подлинности, этот ПИН-код необходимо будет вводить в мобильном приложении или в ходе каждого телефонного вызова для проверки подлинности совместно с одноразовым секретным кодом (OTP) в ответ на каждое SMS.  Системный администратор мог настроить правила использования ПИН-кода, в частности, его минимальную длину.

 

Серийный номер

Введите серийный номер, напечатанный на тыльной стороне токена OATH.

 

Код

Введите код, отображаемый на токене OATH.

 

 

Чтобы вернуться на страницу Вход, нажмите кнопку Отмена.
